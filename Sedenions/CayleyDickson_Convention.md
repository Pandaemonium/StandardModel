# Cayley-Dickson Convention for the Sedenion Track

This document fixes the convention for the sedenion zero-divisor project.
It intentionally differs from the older project octonion convention in
`PhysicsSM.Algebra.Octonion.Basic`.  That is acceptable, but the two
conventions must never be silently mixed.

## Basis Order

Use four-bit labels:

```text
abcd <-> i^d j^c ell^b m^a
```

These are basis names, not freely associative products.  The ordered basis is:

```text
0000 = 1
0001 = i
0010 = j
0011 = ij
0100 = ell
0101 = i ell
0110 = j ell
0111 = ij ell
1000 = m
1001 = i m
1010 = j m
1011 = ij m
1100 = ell m
1101 = i ell m
1110 = j ell m
1111 = ij ell m
```

The product index is always XOR:

```text
e_a e_b = +/- e_(a xor b).
```

The signs are determined recursively by the Cayley-Dickson pair rule.

## Pair Rule

Write a sedenion as an octonion pair:

```text
(a,b) = a + b m.
```

Use:

```text
(a,b)(c,d) = (a c - conjugate(d) b, d a + b conjugate(c)).
```

For old octonion basis units `q,r`, this gives:

```text
q * r       = q r
q * (r m)  = (r q) m
(q m) * r  = q conjugate(r) m
(q m)(r m) = - conjugate(r) q
```

For imaginary octonion basis units `r != 1`, the last two simplify to:

```text
(q m) * r  = -(q r) m
(q m)(r m) = r q
```

The special case `r = 1` must use the pair rule.

## Octonion Reference Triples

In the three-bit octonion subalgebra, use the positive triples:

```text
(001,010,011)
(001,100,101)
(001,111,110)
(010,100,110)
(010,101,111)
(011,100,111)
(011,110,101)
```

These are exactly the triples generated by the recursive Cayley-Dickson rule
from:

```text
1, i, j, ij, ell, i ell, j ell, ij ell.
```

The diagonal lines are "backwards" because:

```text
q * (r ell) = (r q) ell.
```

For example:

```text
i * (ij ell) = j ell,
```

so one diagonal is:

```text
i -> ij ell -> j ell -> i.
```

## Fano Orientation Consistency

XOR gives the seven unoriented Fano lines.  A proposed orientation is
octonion-consistent if its flip vector relative to the reference orientation is
a `[7,4,3]` Hamming codeword.

With flip bits `b_1,...,b_7` in the reference order above, the parity checks
are:

```text
b_4 + b_5 + b_6 + b_7 = 0
b_2 + b_3 + b_6 + b_7 = 0
b_1 + b_3 + b_5 + b_7 = 0
```

over `F_2`.

Equivalently, a legal orientation differs from the reference orientation by a
basis sign change:

```text
e_a -> epsilon_a e_a,   epsilon_a in {+1,-1}.
```

For a line `{a,b,c}` with `a xor b = c`, the line flips exactly when:

```text
epsilon_a epsilon_b epsilon_c = -1.
```

The legal flip patterns form the length-7 Hamming code.

## Relationship to Existing Project Convention

The existing `PhysicsSM.Algebra.Octonion.Basic` convention is anchored by
different products, for example:

```text
e011 * e111 = e100
```

The convention in this folder is the recursive Cayley-Dickson convention.  It
is more natural for the sedenion branch because it extends directly from:

```text
R -> C -> H -> O -> S.
```

Any bridge to existing octonion code must therefore be explicit and sign
corrected, analogous to the existing Cayley-Dickson/octonion convention bridge.
