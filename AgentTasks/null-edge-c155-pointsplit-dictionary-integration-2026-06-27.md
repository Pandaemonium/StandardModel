## 1. How point-splitting implements flavored mass and species selection (imported)

### 1.1 The species problem

A naive lattice Dirac operator
`D_naive(p) = (i/a) Σ_μ γ_μ sin(a p_μ)`
has `2^d` zeros in the Brillouin zone, one at each **corner** `p_μ ∈ {0, π/a}`. These `2^d` degenerate modes are the *species doublers*. For minimally doubled fermions the count is reduced (to 2) by an explicit anisotropy, but the same corner-indexed degeneracy structure governs the doublers. The doublers organise into a *taste* (flavor) algebra generated by products of γ-matrices.

### 1.2 The Wilson term lifts everything; a flavored mass lifts *selectively*

The standard Wilson term `W(p) = (r/a) Σ_μ (1 − cos a p_μ)` vanishes at the origin corner and is positive at every other corner, giving all `2^d − 1` doublers a large mass and leaving one physical fermion. This is a **constant-sign** lift: every doubler is pushed the same way.

A **flavored mass term** is instead engineered to take *different signs/values at different corners*, so it can lift a chosen subset of doublers while leaving a chosen flavor multiplet light. The papers build it by **point-splitting**: rather than a local mass `m ψ̄(x) ψ(x)`, one uses a bilinear that hops to neighbours,
`ψ̄(x) Γ ψ(x ± μ̂)` symmetrised, whose momentum-space symbol is a product/sum of `cos a p_μ` factors dressed with taste generators.

The canonical fully point-split scalar is
`M(p) = Σ over the hypercube ↦ value ∝ Π_μ cos a p_μ`,
which evaluates at corner `c = (c_1,…,c_d)`, `c_μ ∈ {0,1}` (`0 ↦ 0`, `1 ↦ π`), to

```
M(c) = Π_μ (+1 if c_μ = 0, −1 if c_μ = π) = (−1)^(#{μ : c_μ = π}).
```

So `M` is `+1` at the origin, `−1` at every single-`π` corner, `+1` at every double-`π` corner, …: it grades the doublers by the **parity of the number of `π`-components**.

### 1.3 Species selection via spectral flow

One forms the Hermitian operator
`H(m) = γ5 (D_naive + M_flavored − m)`
and tracks eigenvalue crossings of `H(m)` as `m` is dialed. Each doubler crosses zero at `m` equal to its flavored-mass value `M(c)`. The **net spectral flow** between two mass values counts the topological charge weighted by the species in that mass window. Choosing `m` between the `M(c)` levels selects which flavor multiplet stays light — this is *species selection by the flavored mass*, and it is what makes a naive/minimally-doubled **overlap** operator with a prescribed flavor number, while keeping hypercubic symmetry (the `1011.0761` result).

## 2. Dictionary: point-split taste projectors ↔ null-edge branch/flavor projectors

The null-edge seed is `H_ne = Γ_K (D_ne + W_branch − m0 R)`, `T_br = sign(H_ne)`, `D_ov,ne = ρ (1 + Γ_K T_br)`.

| Point-split / overlap literature | Null-edge branch language | Formal anchor in `C155.lean` |
|---|---|---|
| BZ corner `c ∈ {0,π}^d` (a species doubler) | a **branch** | `Corner d := Fin d → Bool` |
| number of `π`-components of `c` | **branch grading** / species level | `level` |
| one-dimensional point-split factor `cos a p_μ` at a corner | the **per-null-edge branch-Pauli sign** (`+1` / `−1`) | `pauliSign : Bool → ℤ` |
| fully point-split flavored mass `Π_μ cos a p_μ` | the **diagonal of `W_branch`** over branches | `psMass : Corner d → ℤ` |
| taste projector `P = Π_μ (1 ± γ_μγ5…)/2` selecting a corner subset | branch/flavor projector onto `{c : M(c) = +1}` | `{c | psMass c = 1}` (via `psMass_balanced`) |
| flavored mass = tensor product of single-link taste structures | `W_branch` = `⊗_μ` (branch-Pauli seed), one factor per null edge | `psMass_eq_tensor` |
| flavored mass `= +1` at origin doubler | `W_branch` diagonal `= +1` on the trivial branch | `psMass_origin` |
| flavored mass is a sign `±1` per corner | branch weight is a sign | `psMass_abs_one`, `psMass_eq_neg_one_pow` |
| equal-size flavor split (`Σ_c M(c) = 0`) | `W_branch` splits branches into two equal flavor groups | `psMass_balanced` |
| Wilson mass `m`-window between `M(c)` levels selecting a multiplet | choice of `m0 R` between branch weights | (design input; see §4) |

**The single most useful identity** is `psMass_eq_tensor`:

```
psMass c = Π_i pauliSign (c i).
```

It says the point-split flavored mass *is exactly the `d`-fold tensor product of the one-edge branch-Pauli seed*. The literature's flavored mass and the project's branch-Pauli insertion are therefore the **same object at different ranks**: one null edge ↦ one `cos a p_μ` ↦ one Pauli sign.

## 3. Comparison with the C145 branch-Pauli seed and the C138 sign-straddling condition

### 3.1 Versus the C145 branch × flavor × qutrit finite seed

C145 used an **abstract branch-Pauli insertion**: a single Pauli matrix `σ` on a 2-level branch index, of uniform magnitude, flipping sign between exactly two branches. The point-split dictionary tells us this is the **`d = 1` slice** of the general construction:

- `pauliSign` *is* the C145 branch-Pauli sign on one null edge.
- `psMass = ⊗_μ pauliSign` is the C145 seed promoted to the full hypercube of `2^d` branches, with the *graded* sign structure `(−1)^level` rather than a single ad-hoc flip.

So C145 is recovered, and the point-split version supplies (a) the correct number of branches (`2^d`), (b) the correct relative signs (parity grading), and (c) a built-in flavor projector (the even/odd-level split), none of which the abstract seed fixed.

### 3.2 Versus the C138 anticommutation / sign-straddling condition

C138 established that `sign(H_ne)` only transfers across branches when the seed is **sign-straddling** — an *odd* seed alone is not enough; the seed must actually produce opposite signs on different branches so that `H_ne` crosses zero between them. In spectral (diagonal) form this is exactly:

```
∃ branches c, c'  with  weight c > 0  and  weight c' < 0.
```

The dictionary makes this automatic for the point-split mass:

- `psMass_straddle` proves the point-split flavored mass straddles for every `d ≥ 1` (origin gives `+1`, any single-`π` corner gives `−1`).
- `constMass_not_straddle` proves a **constant** Wilson mass can *never* straddle.

This is the precise, machine-checked statement that the C138 condition *fails* for the naive local Wilson mass and *holds* for the point-split flavored mass. The off-diagonal anticommutation `{Γ_K, W_branch}` that C138 phrases operatorially shows up here as the diagonal sign change across branches; the Lean file captures the spectral shadow, and the operator-level anticommutation remains a design hypothesis (§4).

## 4. Lean-friendly interface for a null-edge point-split flavored mass term (deliverable 4)

The interface is `C155.FlavoredMassSeed` in `RequestProject/C155.lean`:

```lean
structure FlavoredMassSeed (d : ℕ) where
  weight   : Corner d → ℤ                         -- diagonal of W_branch over branches
  straddle : ∃ c c', 0 < weight c ∧ weight c' < 0  -- the C138 sign-straddling requirement
```

- The **datum** is the branch-indexed weight, i.e. the diagonal of `W_branch`.
- The **defining property** is sign-straddling, which is exactly the C138 precondition for `T_br = sign(H_ne)` to cross branches.
- The canonical inhabitant `psSeed d (h : 1 ≤ d)` is the point-split flavored mass, built from `psMass` and discharged by `psMass_straddle`.

This is deliberately the *minimal* interface that the sign-transfer machinery consumes. To grow it toward the full operator, the intended extensions (each an explicit, still-to-prove hypothesis, not an a x i o m) are:

1. **Carry the γ/Γ_K structure**: replace `weight : Corner d → ℤ` by a self-adjoint operator on each branch and require `{Γ_K, W_branch} = 0` (the operator form of C138). The diagonal of that operator must reduce to `psMass` — `psMass_eq_neg_one_pow` is the target reduction.
2. **Mass-window selection**: add the choice of `m0 R` and a predicate that it lies strictly between two branch weights, so that the selected light multiplet is `{c : weight c < m0}`; `psMass_balanced` guarantees this can be tuned to an equal split.
3. **Locality**: the point-split mass is finite-range by construction, so it slots directly into the C143/C151 path-shell / HJL locality bridge without the non-ultralocal tail that the overlap kernel itself carries.

## 5. Is this a better `W_branch` than the abstract branch-Pauli insertion? (deliverable 5)

**Yes**, and the file makes the reason concrete rather than rhetorical:

1. **It is the same object, correctly ranked.** `psMass_eq_tensor` shows the point-split mass is the `d`-fold tensor product of the branch-Pauli seed. The abstract C145 insertion was the rank-1 (`d = 1`) special case; the point-split form gives the genuine `2^d`-branch operator with the right relative signs.
2. **It satisfies the C138 gate by construction.** `psMass_straddle` (holds) vs. `constMass_not_straddle` (fails) is a machine-checked proof that the point-split mass straddles where the naive local mass cannot, so it is *guaranteed* to support sign transfer; the abstract insertion only stipulated this.
3. **It comes with a canonical flavor projector.** `psMass_balanced` shows the branches split into two equal flavor groups via `(−1)^level`, supplying the taste/branch projector that the abstract seed left unspecified.
4. **It is finite-range / local**, so it is compatible with the existing C143/C151 locality control, unlike an arbitrary abstract insertion whose range was never fixed.

**Recommendation.** Adopt the point-split flavored mass `psMass = ⊗_μ pauliSign` as the concrete model of `W_branch`, instantiate `FlavoredMassSeed` with it, and carry out the §4 extension (1) to upgrade the diagonal sign-straddling to the operator anticommutation `{Γ_K, W_branch} = 0` that C138 needs in full. This replaces the finite toy seed with a construction tied directly to the naive/minimally-doubled overlap literature.
